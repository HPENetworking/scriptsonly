switchdb help information

switchdb is a mini-sql database used for tracking switches which are managed by Ansible. The fields in the database hold the minimal variables needed to get a swicth online and ready to be managed by ansible. There are a couple ways of performing the Zero Thouch operation. If you are using the process developed by James Delancey then you will want to copy the JSON output to the tftpboot directory on the techops workbench.

The are two basic ways to get the switches from cardboard to production. One is using a home brewed techops process with a Linux host configured with DHCP option 66 and 67. It is also a tftpserver. Comware switches load a python file and execute it locally and create the basic configuration necessary for Ansible management.
The second way is to use IMC.
Using IMC to implement ZTP with this application is currently not possible and scheduled for the next relase.
Current Behavior When the application starts it will have 25 test MAC addresses all ready installed. This allows for test driving the tool before using it. At this release these are the primary functions.

This option will generate a list of all switches in an Ansible YAML format.
